# 고스돕 UI 규칙

## 용어 규칙
- `구역`: 메인보드 안의 큰 덩어리다. 예: 상대 A 구역, 상대 B 구역, 바닥 구역, 더미 구역, 내 구역.
- `칸`: 각 플레이어 구역 안에서 카드 종류별로 나뉜 자리다. 예: 광 칸, 띠 칸, 열끗 칸, 피 칸.
- `상단 정보줄`: 이름, 점수, 선 표시가 있는 줄이다.
- `먹은 패 구역`: 광/띠/열끗/피가 정리되는 영역이다.
- `손패 구역`: 내 손패가 놓이는 영역이다. 상대 손패는 기본적으로 보이지 않는다.
- `헤더`: 페이지 맨 위 제목과 주요 버튼이 있는 영역이다.
- `메인보드`: 초록색 판 전체를 가리킨다.
- `사이드바`: 오른쪽 튜터/코치/로그/룰 패널 영역이다.
- `튜터 카드`: 사이드바 안의 개별 박스 하나를 뜻한다. 예: 전략 카드, 흐름 카드, 위험 카드, 기술 카드.
- `고스톱 창`: 고/스톱 선택 모달을 뜻한다.
- `흔들기 창`: 흔들기 선택 모달을 뜻한다.
- `먹을 패 선택 창`: 둘 중 어떤 패를 먹을지 고르는 모달을 뜻한다.
- `전체 패 참고표`: 월별 화투 4장을 모아 보여주는 참고 모달을 뜻한다.
- 앞으로 UI 수정 대화에서는 `구역 > 칸 > 카드` 순서로 부른다.

## 자리와 공개 정보
- 나는 아래 가운데, 상대 A는 왼쪽, 상대 B는 오른쪽이다.
- 상대 손패는 비공개 요약만 보인다.
- 상대 공개패는 A 시계 90도, B 반시계 90도로 보이게 한다.

## 손패와 추천
- 내 손패는 같은 월끼리 정렬한다.
- 카드는 그림 자체를 눌러 낸다.
- 추천 정보는 손패 아래 배지와 hover로 본다.
- 사이드바에 추천 순위 카드를 길게 늘어놓지 않는다.

## 먹은 패 정리
- 광은 왼쪽, 띠/열끗은 가운데, 피는 오른쪽이다.
- 피는 아래줄부터 5장씩 쌓아 위로 올라가게 정리한다.
- 상대 먹은 패는 다른 칸을 침범하지 않게 자기 자리 안에서 정리한다.
- 상대 손패 장수 표시는 약하게, 먹은 패 정리 영역은 넓게 확보한다.

## 모달과 진행
- 둘 중 하나를 고르는 모달에는 닫기 X가 있어야 한다.
- 닫으면 다른 패를 다시 고를 수 있어야 한다.
- 느리게, 보통, 한 턴씩 보기 모드를 지원한다.
- 초록 판 영역은 사이드바보다 크게 보여 판 읽기에 집중되게 한다.

- 상대 비공개 손패 영역은 기본으로 숨긴다. 상대 판에서 중요한 것은 먹은 패 흐름이므로, 회전된 공개 먹은 패에 공간을 더 준다.
- 상대 먹은 패는 잘리지 않도록 오버플로를 열고, 좌측 상대는 시계 방향 90도, 우측 상대는 반시계 방향 90도로 보이게 유지한다.
- 내 손패 영역은 상대 공개패 가독성을 해치지 않도록 상대보다 약간 더 압축해도 된다.

- 패를 낼 때와 가져올 때는 짧은 카드 모션을 준다. 내 자리는 아래에서 올라오고, 좌측 상대는 왼쪽에서, 우측 상대는 오른쪽에서 들어오는 느낌을 준다.
- 뒤집을 패를 받을 때는 덱 더미가 짧게 반응하고, 먹은 패로 들어간 카드는 과하지 않게 한 번 튀어오른다.
- 룰 기준 패널은 아래쪽에 접어두고, 필요할 때만 펼쳐 보게 한다. 기본 시선은 튜터와 해설 로그에 가야 한다.
- 코치와 상의 패널을 두어, 사용자가 지금 판 기준으로 질문하고 답을 받게 한다.
- 룰 기준 접힘 패널은 화살표와 펼치기/접기 문구로 상태를 바로 알 수 있어야 한다.
