# 고스돕 튜터 게임 규칙 기준서

이 문서는 현재 프로젝트의 단일 기준 문서다.
README.md의 표준룰 원칙을 기본으로 하고, 대화에서 합의한 UX 요구사항과 옵션룰은 따로 분리한다.

## 1. 프로젝트 목표

- 이 게임은 대전용 고스돕이 아니라 초보자용 고스돕 튜터 게임이다.
- 사용자는 매 턴마다 왜 그 선택이 유리한지 설명을 받아야 한다.
- 카드 추천 표시는 임의 숫자보다 1순위, 2순위처럼 직관적인 순위 표기가 우선이다.
- 설명은 규칙 용어보다 전략 언어를 우선한다.
- 마우스오버 시 해당 카드 추천 이유를 바로 볼 수 있어야 한다.
- 예: 점수패라는 말만 쓰지 말고, 지금 챙기면 상대에게 안 넘어간다고 설명한다.

## 2. 룰 계층

### 2-1. 기본룰

- 한국에서 통용되는 표준 고스돕 룰을 우선한다.
- 광, 열끗, 띠, 피 분류는 표준룰 기준으로 계산한다.
- 비광 포함 3광은 2점, 일반 3광은 3점, 4광은 4점, 5광은 15점으로 계산한다.
- 열끗 5장부터 1점, 이후 1장 추가마다 1점이다.
- 띠 5장부터 1점, 이후 1장 추가마다 1점이다.
- 피 10장부터 1점, 이후 1장 추가마다 1점이다.
- 쌍피는 피 2장 가치다.
- 홍단, 청단, 초단은 각 3점이다.
- 고도리는 새 그림 열끗 3장 조합으로 5점이다.
- 손패 1장을 내고, 같은 월을 가져가고, 더미에서 1장을 뒤집어 같은 방식으로 처리한다.
- 바닥에 같은 월 1장이면 그 카드와 가져간다.
- 바닥에 같은 월 2장이면 1장을 선택해 가져가고 1장은 남긴다.
- 바닥에 같은 월 3장이면 해당 월 4장을 모두 가져간다.
- 손패가 모두 떨어지거나, 덱이 끝나거나, 승리 조건에 도달하면 판을 종료한다.
- 시작 손패에 같은 월 4장이 모인 총통은 10점 승리로 처리한다.

### 2-2. 옵션룰

- 조커는 기본룰이 아니라 옵션룰이다.
- 쪽, 쓸, 뻑, 첫뻑, 따닥 피, 싹쓸이 피, 조커 뺏김, 배수, 박 규칙은 옵션룰 또는 로컬룰로 분리한다.
- 옵션룰은 UI에도 기본룰과 분리해서 보여준다.

## 3. 현재 프로젝트 기본 설정

- 인원: 3명
- 자리: 나는 아래 가운데, 상대 A는 왼쪽, 상대 B는 오른쪽
- 턴 방향: 시계 방향
- 선: 전판 승자가 다음 판 선
- 조커를 켠 경우 덱에 조커 2장을 포함한다
- 조커는 UI에서 켜기/끄기 가능하며, 현재 상태를 옵션룰로 표시한다. 조커는 피 1장 가치로 계산한다

## 4. 대화에서 합의된 UX 규칙

### 4-1. 판 표시

- 내 손패만 실제 카드로 보여준다.
- 내 손패는 같은 월끼리 모아 보이도록 정렬한다.
- 상대 손패는 공개하지 않는다.
- 상대 손패는 카드 그림 대신 손패 장수 정도만 보여주는 방향이 좋다.
- 상대 A의 공개 정보 영역은 내가 보는 시점 기준 시계 방향 90도 느낌으로 보이게 한다.
- 상대 B의 공개 정보 영역은 내가 보는 시점 기준 반시계 방향 90도 느낌으로 보이게 한다.
- 바닥패는 실제 화투 그림으로 보여준다.
- 바닥패에 마우스를 올리면 몇 월인지, 그 줄에 광/띠/열끗/피가 무엇이 있는지 알려준다.

### 4-2. 먹은 패 정리

- 광은 가장 왼쪽에 정리한다.
- 띠와 열끗은 가운데에 정리한다.
- 피는 오른쪽에 정리한다.
- 피는 5장 단위로 줄을 나눠 정리한다.
- 먹은 패는 살짝 겹쳐서 정리한다.
- 상대가 먹은 패는 바닥패 영역과 시각적으로 겹치면 안 된다.
- 각 플레이어가 먹은 패는 자기 자리 안쪽에서만 정리되게 한다.
- 겹침은 너무 과하지 않게, 한 장의 왼쪽이 약 30% 보이도록 한다.

### 4-3. 진행 UX

- 느리게 보기, 보통, 한 턴씩 보기 모드를 지원한다.
- 내 손패는 아래 추천 버튼을 누르는 방식보다 카드 그림 자체를 눌러 내는 방식이 우선이다.
- 한 턴씩 보기에서는 다음 차례가 누구인지 분명히 보여준다.
- 선이 누구인지 항상 보여준다.
- 턴 방향도 항상 보여준다.

### 4-4. 설명 UX

- 문구는 초보자에게 바로 이해되는 전략 언어여야 한다.
- 예: 중요한 패를 지금 챙기면 상대에게 넘겨주지 않는다고 설명한다.
- 홍단, 청단, 초단, 고도리가 이어지고 있는지, 깨졌는지, 상대가 어떤 줄을 노릴 수 있는지 공개 정보 기준으로 알려준다.

## 5. 겸용 카드 처리

- 현재 프로젝트에서는 9월 국화 술잔만 겸용 패 후보로 본다.
- 다만 이 부분은 집룰 차이가 있을 수 있으므로 설정값으로 분리하는 것이 좋다.
- 겸용 패는 잡았을 때 열끗으로 둘지, 쌍피로 둘지 선택 가능해야 한다.
- 선택 결과는 점수 계산과 먹은 패 정리 위치에 반영되어야 한다.

## 6. 조커 처리 원칙

- README 기준으로 조커는 옵션룰이다.
- 조커를 켠 경우 덱에 조커 2장을 포함한다.
- 시작 바닥패에 조커가 깔리면 선이 먼저 가져간다.
- 현재 프로젝트에서는 조커를 피 정리와 점수 계산에서 일반 피 1장 가치로 본다.
- 즉 조커는 쌍피처럼 2점으로 세지 않으며, 피 줄도 `피 값 5` 기준에 맞춰 일반 피 1장처럼 묶는다.
- 손에서 조커를 냈을 때 추가 처리나 초반 손해 규칙은 표준룰이 아니라 집룰/옵션룰로 따로 관리한다.

## 7. 아직 확정되지 않은 항목

- 쪽의 정확한 발동 조건과 피 이동
- 쓸의 정확한 발동 조건과 피 이동
- 뻑과 첫뻑의 처리 방식
- 따닥 피 가져오기 적용 여부
- 싹쓸이 피 가져오기 적용 여부
- 흔들기와 폭탄의 배수
- 고/스톱 세부 규칙
- 피박, 광박, 고박, 멍박 적용 여부
- 조커 옵션룰 상세 처리

## 8. 리뉴얼 방향

- 현재 구현은 여러 차례 수정되며 로직과 UI가 섞여 있어 재정비가 필요하다.
- 다음 리뉴얼은 이 문서를 기준으로 ruleConfig 중심 구조로 다시 나눈다.
- 기본룰과 옵션룰을 코드와 UI에서 명확히 분리한다.
- 튜터 문구는 규칙 설명과 전략 설명을 분리한다.
- 판정 로직과 화면 배치를 분리해서 유지보수 가능하게 만든다.


## 9. 판단 알고리즘 기준

- 나와 AI 모두 자기에게 가장 유리한 선택을 우선한다.
- 판단은 즉시 점수, 단/고도리 연결, 상대 좋은 패 차단, 바닥에 위험한 월 남기지 않기를 함께 본다.
- 먹을 수 없는 턴에는 무조건 낮은 패를 버리는 식이 아니라, 상대에게 덜 열리는 월을 추천해야 한다.
- 둘 중 하나를 고를 때는 카드 종류만 말하지 말고, 현재 몇 점에서 몇 점으로 오르는지와 어떤 줄이 완성되는지 설명한다.
- 고/스톱은 사용자가 직접 선택하되, 현재 점수, 남은 패 수, 다음 좋은 수 존재 여부를 기준으로 추천을 붙인다.
- 흔들기 가능 월이 있으면 버튼으로 선택권을 주고, 지금 흔드는 것이 좋은지 보류가 좋은지도 가이드한다.
- 상대 A 공개영역 카드는 시계 방향 90도, 상대 B 공개영역 카드는 반시계 방향 90도로 보이게 한다.

- 위험도는 현재 점수만이 아니라 공개된 광, 단, 열끗, 피 누적과 조합 진행도까지 함께 보고 판단한다.
- 9월 국화 술잔은 점수 차이가 같으면 열끗보다 쌍피 쪽 배치가 우선이다.

- 상대도 흔들기 조건이 되면 먼저 흔들 수 있어야 하며, 선언 시 로그와 튜터 해설에 즉시 반영한다.

- 상대 공개패는 카드 몇 장만 개별 회전하는 것이 아니라, 공개패 묶음 전체가 A는 시계 90도, B는 반시계 90도로 확실히 보이게 한다.

- 튜터 문구는 "덜 아프다" 같은 감정형 표현보다, 왜 상대 연결이 적은지 근거를 짧게 설명하는 쪽을 우선한다.
- 추천 카드 설명은 개요식으로 1~2개의 핵심 근거만 남기고, 당연한 절차 설명은 생략한다.

- 첫 판 선은 고정이 아니라 랜덤으로 정한다.
- 판 승자는 다음 판 선이 된다.
- 쪽/쓸이 발생하면 상대에게서 피 1장씩 가져오며, 일반피가 있으면 일반피를 먼저 가져간다.

- 추천 1, 2, 3순위는 사이드바에 길게 늘어놓기보다, 손패 아래 배지와 hover 설명으로 확인하는 방식이 우선이다.
- 초록색 판 영역은 사이드바보다 크게 보여 실제 판을 읽는 데 집중되게 한다.
- 피 묶음은 아래 줄부터 5장씩 쌓아 위로 올라가게 정리한다.

- 시작 바닥패에 조커가 깔리면 선이 그 조커를 먼저 가져가고 시작한다.

- 상대 손패 장수 표시는 과하게 강조하지 않고, 공개 정보의 중심은 먹은 패 정리 영역에 둔다.
- 상대 먹은 패 정리 영역은 손패 요약보다 더 넓은 공간을 확보한다.

- 둘 중 하나를 고르는 모달은 닫기 X가 있어야 하며, 닫으면 다른 패를 다시 고를 수 있어야 한다.

- 사이드바에서는 추천 순위 카드를 별도로 보여주지 않고, 추천 정보는 손패 아래 배지와 hover 설명으로만 확인한다.
