# 이번 라운드에서 보류/거절한 항목

## 1. 런타임 안에 리서치 엔진 넣기
- 거절 이유: 모바일 게임에서 불필요하게 무거워진다.
- 대신: `research/` + `harness/` 정적 검사로 분리한다.

## 2. 모든 설명을 장문 코치형으로 바꾸기
- 거절 이유: 실제 플레이 중에는 체크리스트형 경고가 더 빨리 읽힌다.
- 대신: 짧은 체크리스트 + 필요한 경우 채팅형 상세 답변을 병행한다.

## 3. 세로 모바일 플레이 허용
- 거절 이유: 메인보드, 상대 공개패, 내 손패를 동시에 보기 어렵다.
- 대신: 모바일/태블릿은 가로 전용 안내를 강화한다.
